# Regras do projeto

1. **Studio** pode ter testes, presets, universo autoral e módulos avançados.
2. **Client** é neutro, direto e sem vestígios de projetos anteriores.
3. **Print** é separado para gráfica, briefing, layout, orçamento e pré-impressão.
4. **API** é motor interno. Não é produto nem linguagem principal para o usuário final.
5. Novo projeto no Client começa vazio. Presets só entram por ação manual do usuário.
